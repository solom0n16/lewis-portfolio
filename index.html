02 · HOW I WORK

From unclear request to working system.

My process is built for stakeholder reality: unclear asks, messy data, changing requirements, and outputs that still need to ship.

01
Clarify the Decision
Define the business question, decision owner, and what action the analysis should trigger.
02
Map Stakeholders
Identify who uses the output, what they already track, and where reporting friction exists.
03
Audit Data Sources
Check exports, APIs, tables, field definitions, refresh paths, and data quality risks before building.
04
Clean & Model
Turn messy inputs into a reliable model: keys, relationships, measures, validations, and edge cases.
05
Build the Output
Create the dashboard, tracker, or automation around the user journey — not around chart decoration.
06
Validate With Users
Test totals, filters, logic, and usability with the people who will depend on it.
07
Deploy & Monitor
Ship, document refresh behavior, watch failures, and keep improving based on real usage.
